<objective>
Implement the file chunker that splits large files into overlapping, logically-bounded chunks ready for LLM consumption.

Purpose: LLM context windows have limits. The chunker ensures every file — regardless of size — is broken into digestible pieces that preserve semantic coherence. Splitting on logical boundaries (function/class declarations) rather than arbitrary line counts means each chunk is a meaningful unit the LLM can reason about. Overlap ensures no context is lost at chunk boundaries.
Output: A fully implemented core/chunker.ts that replaces the stub, with language-agnostic boundary detection and guaranteed forward progress.

         /^def\s+\w+/,
       ];
       ```

         const trimmed = line.trimStart();
         return BOUNDARY_PATTERNS.some((pattern) => pattern.test(trimmed));
       }
       ```
       - Tests trimmed line (handles indented methods, nested classes)

    6. Implement findBoundaryBefore helper:
       ```typescript
       function findBoundaryBefore(lines: string[], targetIdx: number, minIdx: number): number {
         // Search backward from targetIdx for a logical boundary
         for (let i = targetIdx - 1; i >= minIdx; i--) {
           if (isLogicalBoundary(lines[i])) {
             return i;
           }
         }
         // Fallback: look for a blank line
         for (let i = targetIdx - 1; i >= minIdx; i--) {
           if (lines[i].trim() === '') {
             return i + 1; // start new chunk AFTER the blank line
           }
         }
         // No boundary found — return targetIdx (raw cut)
         return targetIdx;
       }
       ```
       - First pass: look for function/class declaration
       - Second pass: look for blank line (chunk starts after blank line)
       - Fallback: cut at the raw target index
       - minIdx prevents searching too far back (enforces meaningful chunk size)

    7. Implement the main chunkFile function:
       ```typescript
       export function chunkFile(content: string, filePath: string): Chunk[] {
         // Handle empty content
         if (!content || content.trim() === '') {
           return [];
         }

         const lines = content.split('\n');
         // Remove trailing empty line from files ending with newline
         if (lines.length > 0 && lines[lines.length - 1] === '') {
           lines.pop();
         }

         const totalLines = lines.length;

         // Empty after trimming trailing newline
         if (totalLines === 0) {
           return [];
         }

         // Single chunk case: file fits in one chunk
         if (totalLines <= CHUNK_SIZE) {
           return [
             {
               filePath,
               startLine: 1,
               endLine: totalLines,
               chunkIndex: 0,
               content: lines.join('\n'),
             },
           ];
         }

         // Multi-chunk case
         const chunks: Chunk[] = [];
         let start = 0;
         let chunkIndex = 0;

         while (start < totalLines) {
           let end = Math.min(start + CHUNK_SIZE, totalLines);

           // If not the last chunk, try to snap to a logical boundary
           if (end < totalLines) {
             // Only snap if we'd still make meaningful progress (at least 1/4 of CHUNK_SIZE)
             const minEnd = start + Math.floor(CHUNK_SIZE / 4);
             const snapped = findBoundaryBefore(lines, end, minEnd);
             if (snapped > minEnd) {
               end = snapped;
             }
           }

           chunks.push({
             filePath,
             startLine: start + 1,      // 1-indexed
             endLine: end,               // 1-indexed, inclusive
             chunkIndex,
             content: lines.slice(start, end).join('\n'),
           });

           chunkIndex++;

           // Calculate next start with overlap
           const nextStart = end - OVERLAP;

           // CRITICAL: guarantee forward progress to prevent infinite loops
           if (nextStart <= start) {
             start = end;
           } else {
             start = nextStart;
           }
         }

         return chunks;
       }
       ```

       KEY IMPLEMENTATION DETAILS:
       - Empty/whitespace-only content returns []
       - Trailing empty string from split('\n') on newline-terminated files is removed
       - Single-chunk fast path avoids unnecessary boundary logic
       - startLine is 1-indexed: start + 1 (0-indexed start becomes 1-indexed)
       - endLine is 1-indexed inclusive: end (since lines.slice(start, end) is exclusive, the last included 0-indexed line is end-1, which is end in 1-indexed)
       - content uses lines.slice(start, end) — standard exclusive end for slice
       - Forward progress guaranteed: if overlap would push nextStart back to or before start, skip overlap
       - minEnd in findBoundaryBefore prevents snapping too far back (chunk must be at least 1/4 of CHUNK_SIZE)
